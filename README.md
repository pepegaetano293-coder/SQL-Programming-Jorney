# SQL Corporate Database Practive

Progetto pratico di SQL focalizzato sulla progettazione, il popolamento e l'interrogazione di un database aziendale inventato.
Tutti gli scripit sono stati sviluppati e testati utilizzando **Bekeeper Studio**.

## Schema del Database
Il database simula la gestione di un'azienda attraverso tre entità principali collegate tra loro:
- **Dipendenti**: Gestione anagrafica, stipendi e data di assunzione.
- **Dipartimenti**: Organizzazione dei settori aziendali e relative sedi.
- **Vendite**: Registro delle transazioni commerciali effettuate dai dipendenti.

## Cosa ho imparato
Durate questo progetto ho approfondito i seguenti congetti SQL:
- **DDL (Data Definition Language)**: Creazione di tabelle con vincoli di integrità.
- **DML (Data Manipulation Language)**: Inserimento dati e aggiornamenti.
- **Joins**: Utilizzo di 'INNER JOIN' per collegare dipendenti ai propri dipartimenti e vendite.
- **Aggregazioni**: Uso si 'SUM()', 'AVG()', 'COUNT()' e filtri avanzati con 'GROUP BY' e 'HAVING'.
- **Data Management**: Filtro delle transazioni basato sull'anno tramite la funzione 'YEAR()'.

## Strumenti utilizzati
- **Client**: Bekeeper Studio.
- **Database Engine**: MariaDB / MySQL.
